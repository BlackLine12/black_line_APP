# App Audit — BlackLine
**Date:** 2026-05-01

## Executive Summary

BlackLine has a functional skeleton for both CLIENT and STUDIO flows, but suffers from three systemic problems. First, **security policy is violated at the foundation**: JWT tokens are stored in `localStorage` (not `sessionStorage`), directly contradicting the CLAUDE.md rule that mandates session storage. Second, the **Match flow has a hidden breakage** — artists configure their city via a free-text input, but the client Match page uses a strict catalog; any artist who types "GDL", "Guadalajara, Jal.", or any variant of the catalog name becomes invisible in searches. Third, several **screens are informational dead-ends** with no actionable next step for the user: the artist dashboard shows pending counts but offers no approve/reject action, and Mis Citas shows a pending health consent with no way to complete it.

The CLIENT flow from Quote → Match → Appointment works end-to-end but has gaps that strand users mid-journey. The STUDIO flow is more fragmented: the dashboard, agenda, and portfolio are separate silos with no continuity between them. The most urgent fix is the Match city mismatch — it renders the entire matchmaking feature unreliable in production.

## Frontend ↔ Backend Gaps

**Endpoints with no frontend consumer:**
- `GET /api/quotes/appointments/<pk>/` — `AppointmentDetailView` exists but no component calls it
- `GET /api/quotes/appointments/<pk>/health-consent/` — Read endpoint exists, never called
- `GET /api/artists/profiles/` — Full artist list exists; landing page shows static content
- `GET /api/artists/profiles/<pk>/` — `ArtistService.getProfile(id)` is defined but never called from any component

**Frontend calls without a working route:**
- `navigate(['/artistas', id])` in `studio/dashboard.component.ts` — no matching route in `app.routes.ts`

**Security violation:**
- `TokenStorageService` uses `localStorage` for JWT tokens — CLAUDE.md mandates `sessionStorage`
- `AuthService` uses `localStorage` for `bl_user` — same policy violation

---

## Priority Recommendations

1. **Fix JWT/user storage (Critical, 1h)** — Migrate `TokenStorageService` + `auth.service.ts` from `localStorage` to `sessionStorage`. Affects every user.
2. **Add city autocomplete to studio profile (Critical, 3h)** — Reuse `cities-mx.ts` + the pattern already built in `match.component`. Without this, Match is unreliable.
3. **Add health consent form to Mis Citas (Critical, 4h)** — POST health consent inline modal; read existing consent with GET endpoint.
4. **Remove or implement `/artistas/:id` (Critical, 2h to remove / M to build)** — Easiest fix: remove the "Ver Perfil Público" button. Proper fix: build the public profile page.
5. **Add reject counter-offer button (Medium, 1h)** — One-line backend call, one button in template.
